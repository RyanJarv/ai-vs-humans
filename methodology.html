How we estimate AI vs human code

Overview

There is no perfect, universally accepted measure of how much code hosted on GitHub is authored by AI systems versus humans. This project publishes a transparent estimate, with its assumptions and limits made clear.

Computation

  • Data source: data/estimate.json, updated daily by a GitHub Actions workflow that samples public code via the GitHub API.
  • Metric: estimated share (percentage) of AI‚Äëauthored lines in the sample, plus absolute counts used for the calculation.
  • Heuristic: files explicitly containing phrases like ‚Äúgenerated by ChatGPT/Copilot/Claude‚Äù are counted as AI‚Äëlikely. Typical code files (excluding those phrases) are counted as human‚Äëlikely. This underestimates AI‚Äëauthored code.

Limitations

  • Detection: most AI coding tools do not annotate code; explicit markers are rare, so the result undercounts AI‚Äëauthored code.
  • Coverage: API rate limits and sampling strategies introduce uncertainty when extrapolating to all public repositories.
  • Scope: this estimate focuses on public GitHub code and does not include private repositories or other platforms.
