Methodology

How we estimate AI vs human code

Overview

There is no perfect, universally accepted measure of how much code hosted on GitHub is authored by AI systems versus humans. This project publishes a transparent estimate, with its assumptions and limits made clear.

Computation

  • Data source: a JSON file in this repository at data/estimate.json. It is intended to be updated periodically by a GitHub Actions workflow using public GitHub APIs and documented heuristics.
  • Metric: we display an estimated share (percentage) of AI-authored lines-of-code across public repositories, plus absolute counts used for the calculation.
  • Defaults: until the automated pipeline is finalized, values are placeholders you can update by editing data/estimate.json.

Limitations

  • Detection: most AI coding tools do not annotate code with reliable markers, making precise attribution difficult without repository-level analysis and opt-in signals.
  • Coverage: API rate limits and sampling strategies introduce uncertainty when extrapolating to all public repositories.
  • Scope: this estimate focuses on public GitHub code and does not include private repositories or other platforms.

Contributing

Suggestions for better signals or data sources are welcome. Open an issue or PR in the GitHub repository linked above.
