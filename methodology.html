How we estimate AI vs human code

Overview

There is no perfect, universally accepted measure of how much code hosted on GitHub is authored by AI systems versus humans. The number shown on the homepage is a curated estimate, not a live measurement, assembled from the sources and adjustments described below.

Source inputs

  • Baseline study. Daniotti et al. (2025) analyse 80 million public Python commits on GitHub and attribute 30.1 % of functions authored in the United States to AI systems.
    arXiv:2506.08945
  • Platform telemetry. GitHub reports that Copilot now generates 46 % of the code typed by engaged users, highlighting the upper bound for heavy-adopter teams.
    Visual Studio Magazine (Feb 2023)
  • Adoption signal. Stack Overflow‚Äôs 2024 developer survey finds 44 % of professional developers use AI coding tools at least weekly, indicating partial but not universal uptake.
    Stack Overflow Developer Survey 2024

Headline calculation (25.7 % AI share)

The displayed percentage is a judgemental combination of the inputs above. We apply two explicit discounts to the Daniotti et al. baseline to account for cross-language variation and incomplete adoption outside the studied cohort:

  • Language coverage adjustment: multiply 30.1 % by 0.90 to reflect languages with lower Copilot usage than Python, guided by the spread of adoption in GitHub telemetry.
  • Adoption adjustment: multiply by 0.95 so that the final figure better matches the share of developers using AI tools weekly (‚âà44 %).

Calculation: 30.1 % √ó 0.90 √ó 0.95 ‚âà 25.7 %. We scale this ratio to illustrative absolute counts in data/estimate.json (6.2 billion AI-attributed lines versus 17.9 billion human-attributed lines, totalling 24.1 billion lines).

Repository data files

  • data/estimate.json: manually curated figure shown on the site (updated 17 Oct 2025 at 16:00 UTC).
  • data/estimate_action.json: heuristic output from the scheduled GitHub Action (scripts/update-estimate.js), useful for tracking trend direction but not surfaced publicly.

Limitations

  • The baseline study covers Python commits from U.S. contributors; applying it globally and across languages introduces unquantified error.
  • Platform telemetry is self-reported and skewed toward teams already investing in AI tooling, so real-world averages may be lower.
  • The Action-based sampler relies on explicit phrases ("generated by...") and therefore substantially undercounts AI-authored code.
